# 提交类型

类型 说明
feat 新功能
fix 修复错误
docs 文档修改
style 格式修改（不影响代码运行的变化，如空白、格式化、缺少分号等）
refactor 重构（既不新增功能，也不是修改错误的代码变动）
test 测试（添加缺失测试或更正现有测试）
chore 构建过程或辅助工具的变动
perf 改进性能的代码更改
